## Project Overview

Docusaurus v3 (3.9.2) site using the classic preset, TypeScript, and React 19. Uses npm as the package manager (not yarn, despite the README).

## Commands

- `npm start` - Start local dev server with hot reload
- `npm run build` - Production build to `build/` directory
- `npm run serve` - Serve the production build locally
- `npm run typecheck` - Run TypeScript type checking (`tsc`)
- `npm run clear` - Clear Docusaurus generated files (`.docusaurus/`)

## Architecture

- **`docusaurus.config.ts`** - Main site config: metadata, presets, navbar, footer, theme. Uses `future.v4: true` for Docusaurus v4 compatibility.
- **`sidebars.ts`** - Sidebar configuration. Currently uses `autogenerated` from the `docs/` folder structure.
- **`docs/`** - Documentation content as Markdown/MDX. Subdirectory ordering controlled by `_category_.json` files.
- **`blog/`** - Blog posts. Supports `authors.yml` and `tags.yml` for metadata.
- **`src/pages/`** - Standalone pages (React TSX or Markdown). Each file becomes a route.
- **`src/components/`** - Reusable React components (e.g., `HomepageFeatures`).
- **`src/css/custom.css`** - Global CSS overrides and Docusaurus CSS variable customization.
- **`static/`** - Static assets served at site root (images, favicon).

## Key Conventions

- Config files (`docusaurus.config.ts`, `sidebars.ts`) run in Node.js, not the browser.
- Path alias `@site/` resolves to the project root in imports.
- CSS uses CSS Modules (`.module.css` files) for component-scoped styles.
- `onBrokenLinks: 'throw'` is set, so builds will fail on broken internal links.
